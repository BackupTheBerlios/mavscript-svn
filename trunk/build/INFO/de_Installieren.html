Mavscript installieren

Kurzanleitung

  1. Das Zip-Archiv an beliebige Stelle entpacken, z.B. nach C:\Programme (Windows) resp. /usr/local/share (Linux). Der Installationspfad, auf den in den nächsten Punkten verwiesen wird, heisst dann entsprechend C:\Programme\mavscript.

  2. Die Datei Installationspfad\INFO\desktoplinks\Mavscript.windows.lnk resp. Installationspfad/INFO/desktoplinks/Mavscript.linux.desktop auf den Desktop kopieren.

  3. Mit der rechten Maustaste auf Eigenschaften, im Dialog gegebenenfalls den Installationspfad anpassen!

  4. Falls noch nicht vorhanden, muss Java (Java Runtime Environment 1.4 oder höher) installiert werden.


Windows (2000, XP)

  1. Das Zip-Archiv mavscript*.zip in ein beliebiges Verzeichnis entpacken. Ein gängiger Ort dafür ist C:\Programme. Es wird ein Unterverzeichnis mit dem Namen mavscript erstellt. Der Installationspfad lautet also C:\Programme\mavscript.

  2. Java (JRE) in der Version 1.4 oder höher muss installiert sein. Dies kann überprüft werden, indem man die Eingabeaufforderung (Command Prompt, resp. DOS-Fenster) öffnet und dort den folgenden Befehl eingibt: java -version Falls die Meldung kommt, java sei nicht gefunden worden, ist es nicht installiert.

  3. Im Verzeichnis C:\Programme\mavscript\INFO\desktoplinks befindet sich eine Datei Mavscript.windows.lnk. Diese Datei - bei gedückter Ctrl-Taste - mit der Maus auf den Desktop ziehen.
    Das soeben erzeugte Symbol auf dem Desktop markieren, die rechte Maustaste drücken und den Eigenschafts-Dialog öffnen. In einem der Reiter hat es ein Befehlsfeld. Dort sollte stehen: C:\Programme\mavscript\mavscript-yacas.bat . Im Feld Arbeitsverzeichnis: C:\Programme\mavscript

  4. Falls alles geklappt hat, ist Mavscript nun installiert. Im Verzeichnis C:\Programme\mavscript\examples hat es Beispieldateien. Mit der Maus eine solche (z.B. 1-yacas.txt) auf das in Punkt 3 erstellte Mavscipt-Symbol auf dem Desktop ziehen. Es öffnet sich ein Fenster und, wenn alles geklappt hat, steht auf der letzten Zeile "Fertig. Mavscript wird beendet." Die Ausgabedatei out.1-yacas.txt wurde erzeugt.
  5. Nicht zwingend erforderlich, jedoch nützlich um formatierten Text zu erzeugen, ist OpenOffice. Die OpenOffice-Writer Dateien mit der Endung .odt oder .sxw können genauso wie Text- oder html-Dateien mit Mavscript verwendet werden. Microsoft Word Dateien (Endung .doc) funktionieren nicht. Wer jedoch Word kennt, kommt auch gleich mit OpenOffice zurecht.

Es funktioniert nicht...

  • Falls Mavscript schon gar nicht startet, sind vielleicht die Pfadangaben (siehe oben) falsch.

  • Um die Fehlerquelle zu finden, geht man wie folgt vor: DOS-Fenster öffnen.
    cd \Programme\mavscript
    mavscript examples/1-yacas.txt eingeben.

  • Enthält die Vorlage-Datei (oder deren Verzeichnisname) ein Leerzeichen, muss ihr Name mit "Gänsefüsschen" umklammert werden. Der direkte Aufruf vom DOS-Fenster aus sieht dann so aus:
    java -jar mavscript*.jar "vorlagedatei mit Leerzeichen"

  • Funktioniert die Option --port nicht, verhindert möglicherweise eine Firewall die Verbindung von Mavscript zum externen Programm. Falls dies der Fall ist, sollte im Mavscript-Fenster eine Meldung darauf hinweisen, dass keine Verbindung erstellt werden konnte. Im Firewallprogramm diese Verbindung zulassen.


LINUX mavscript*.rpm

  1. Java (JRE) in der Version 1.4 oder höher muss installiert sein. Dies kann in der Shell überprüft werden: java -version .

  2. Konsole (Shell) öffnen und dann
    sudo rpm -Uvh mavscript*.rpm
    Root-Passwort eingeben.

  3. Mavscript ist jetzt installiert. Das Programm kann aus einem beliebigen Verzeichnis aufgerufen werden.

    mavscript ./vorlage.txt
    mavscript /home/<user>/vorlage.txt
    mavscript-yacas ./vorlage.txt Abkürzung für
    mavscript --yacas --init /usr/share/mavscript/lib/init.ys ./vorlage.txt
    mavscript-beanshell ./beanshell_vorlage.txt Abkürzung für
    mavscript --beanshell --init /usr/share/mavscript/lib/init.bsh ./beanshell_vorlage.txt

    Die Beispiele befinden sich im Ordner /usr/share/doc/packages/mavscript/examples . Ausprobieren mit
    mavscript-yacas -o ~/out.test.txt /usr/share/doc/packages/mavscript/examples/1-yacas.txt

    Hilfe findet man im Ordner /usr/share/doc/packages/mavscript/INFO oder mit den Befehlen
    mavscript --help sowie
    man mavscript

  4. Um ein KDE-Desktopsymbol zu erstellen, die Datei Mavscript.linux.desktop aus dem Ordner /usr/share/doc/packages/mavscript/INFO/desktoplinks auf den Desktop ziehen und kopieren bestätigen.

    Eine Beispieldatei (z.B. 2-yacas+init.txt) ins home-Verzeichnis kopieren und von dort aus auf Mavscipt-Symbol auf dem Desktop ziehen. Es öffnet sich ein Fenster und, wenn alles geklappt hat, steht auf der letzten Zeile "Fertig. Mavscript wird beendet." Die Ausgabedatei out.2-yacas+init.txt wurde erzeugt.

  5. Nicht zwingend erforderlich, jedoch nützlich um formatierten Text zu erzeugen, ist OpenOffice. Die OpenOffice-Writer Dateien mit der Endung .odt oder .sxw können genauso wie Text- oder html-Dateien mit Mavscript verwendet werden.


LINUX mavscript*.zip

  1. Java (JRE) in der Version 1.4 oder höher muss installiert sein. Dies kann in der Shell überprüft werden: java -version
    Ambitionierte Benutzer können Mavscript auch mit GNU Classpath (>= 0.14) laufen lassen, z.B. mittels JamVM.

  2. Das Zip-Archiv mavscript*.zip entpacken und in ein beliebiges Verzeichnis kopieren. Ein gängiger Ort dafür ist /usr/local/share. Der Installationspfad lautet also /usr/local/share/mavscript. Das schreiben in dieses Verzeichnis erfordert i.d.R. root-Rechte. Schritt für Schritt geht das so: Konsole (Shell) öffnen und dann
    cd /usr/local/share
    sudo unzip /home/USER/mavscript*.zip -d . (Der Punkt nach -d gehört dazu!) Den * nötigenfalls durch die Versionsnummer ersetzen.
    Passwort eingeben.

  3. Ausprobieren aus der Shell
    cd /usr/local/share/mavscript
    ./mavscript    -yacas -o ~/out.test.txt examples/1-yacas.txt

  4. KDE: Im Verzeichnis /usr/local/share/mavscript/INFO/desktoplinks befindet sich eine Datei Mavscript.linux.desktop . Diese Datei mit der Maus auf den Desktop ziehen und kopieren bestätigen.
    Das soeben erzeugte Symbol auf dem Desktop markieren, die rechte Maustaste drücken und den Eigenschafts-Dialog öffnen. Im Reiter Programme im Befehlsfeld sollte stehen: konsole --noclose --notabbar --nomenubar -T "mavscript" --workdir /usr/local/share/mavscript/ -e /usr/local/share/mavscript/mavscript-yacas

  5. KDE: Mavscript ist nun installiert. Im Verzeichnis /usr/local/share/mavscript/examples hat es Beispieldateien. Eine solche (z.B. 2-yacas+init.txt) ins home-Verzeichnis kopieren und von dort aus auf das in Punkt 4 erstellte Mavscript-Symbol auf dem Desktop ziehen. Es öffnet sich ein Fenster und, wenn alles geklappt hat, steht auf der letzten Zeile "Fertig. Mavscript wird beendet." Die Ausgabedatei out.2-yacas+init.txt wurde erzeugt.

  6. Nicht zwingend erforderlich, jedoch nützlich um formatierten Text zu erzeugen, ist OpenOffice. Die OpenOffice-Writer Dateien mit der Endung .odt oder .sxw können genauso wie Text- oder html-Dateien mit Mavscript verwendet werden.

Es funktioniert nicht...

  • Um die Fehlerquelle zu finden, geht man wie folgt vor: In der Shell /usr/local/share/mavscript/mavscript --help eingeben, resp. statt --help eine Vorlagedatei.

  • Funktioniert die Option --port nicht, verhindert möglicherweise die Firewall die Verbindung von Mavscript zum externen Programm. Falls dies der Fall ist, sollte im Mavscript-Fenster eine Meldung darauf hinweisen, dass keine Verbindung erstellt werden konnte. In der Firewallkonfiguration diese interne Verbindung zulassen.

Mac OS X, Solaris und andere Betriebssysteme

Mavscript läuft auf allen Betriebssystemen, auf denen Java (oder GNU Classpath >= 0.14) installiert ist.
  1. Das Zip-Archiv mavscript*.zip entpacken und in ein beliebiges Verzeichnis kopieren.
  2. Ausprobieren aus der Shell
    cd Installationspfad
    ./mavscript    -yacas examples/1-yacas.txt
